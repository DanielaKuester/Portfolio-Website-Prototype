Project 2: Portfolio Website

The Portfolio Website was the first project that involves responsive design. This project contains links to my very first experiments with programming: adapting the layout of an article about IBM's AI Deep Blue to a mockup image, the Animal Trading Card styled with CSS and a Pixel Art Maker created with JavaScript. In the future, I will rework (and hopefully refine) this project and I'll post updates here.

Project 3: Memory Game

The Memory Game starter code provided me with a basic CSS styling and a shuffle method. I made the static website come alive with JavaScript. I especially like the animated cards and I also want to add sounds in future updates. Of all projects, the Memory Game is my favourite project because I learned the first basics of JS debugging with the Chrome Developer tools. What else? I fell in love with the power of event listeners and DOM manipulation.

Project 4: Arcade Game Clone (Frogger)

The lessons leading to the Arcade Game Clone project introduced the principles of object oriented programming and the Frogger clone is the first project where I applied these principles. The encapsulation principle, the separation of concerns and the use of prototypes were big challenges for me, but I'm looking forward to learning more about object oriented programming since it will also help me to improve my ReactJS skills.

Project 5: Feed Reader Testing with Jasmine

For this project, I applied the red to green testing principles to an existing Feed Reader website. The tests include that all articles load and update properly, and the menu changes visibility when the corresponding icon is clicked. It was fascinating to experience how good testing methods can improve the development process when creating a new project. I definitely want to dive deeper into testing!

Project 6: Restaurant Reviews App

The challenges of the Restaurant Reviews App were to fix the given CSS file to make the website responsive, to adhere to modern accessibility standards and to add a service worker for offline use. Implementing the service worker was difficult for me in the beginning, so I followed a tutorial, but I'm planning to write my own version of a service worker in the future.

Project 7: MyReads - A Book Tracking App

The MyReads Book Tracking App is the first of my projects written with ReactJS. It was quite a challenge for me to grasp the basic concepts of React and to apply them to a project, but it turned out a lot better than expected and I even started to like React. I'll probably use some CSS to give the project a more modern look. I also want to remove redundant code so that the project adheres to the DRY principle (Don't Repeat Yourself).

Project 8: Neighborhood Map

The Neighborhood Map was the most challenging Udacity project. The final Nanodegree project didn't include any starter code at all and I had to write it with React. I decided to write most of the project in Vanilla JavaScript because I wanted to find out how the Mapbox API and the Google Maps API work without any JavaScript framework. I'm planning to refine the CSS styling of the existing project and I want to create a new version that uses a library for Mapbox or Google Maps.
